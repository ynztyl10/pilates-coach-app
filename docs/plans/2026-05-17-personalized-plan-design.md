# 智能综合训练计划设计文档

## 1. 背景与目标

现有 PlanGenerator 通过选择知识库条目生成条目列表式的训练计划。教练需要手动从条目中提取训练方向，再根据自己的经验编排每节课的内容。

新功能"智能综合计划"旨在将会员的身体状况描述作为输入，自动从匹配的知识库条目中提取关键信息，去重合并后按优先级重组为一份结构化的综合训练计划。

核心目标：降低教练编排课程的心智负担，同时保证训练逻辑的专业性和安全性。

## 2. 功能定位

在现有 PlanGenerator 旁新增一个"智能综合计划"入口。两者关系：

- 现有模式：选择知识库条目 → 生成条目列表（保留不变）
- 新增模式：输入身体状况描述 → 系统自动匹配条目 → 提取并重组为综合计划

教练可在两个模式间切换使用。

## 3. 输入设计

### 3.1 体态描述

支持两种方式：

1. **标签选择**：从知识库条目名称列表中多选（如"足弓塌陷"、"膝盖内扣"等）
2. **自然语言输入**：教练输入描述文本，系统解析关键词自动匹配

两种输入方式可组合使用——标签选择为主，自然语言为补充。

### 3.2 特殊病史

文本输入框，教练自由填写。常见病史提供快捷标签供一键添加：

- ACL断裂/重建（可细分左右腿+术后时间）
- 肩袖撕裂
- 腰椎间盘突出
- 髋关节置换
- 骨质疏松
- 糖尿病
- 孕产史
- 其他（自由输入）

系统根据病史自动叠加对应的禁忌症和注意事项。

## 4. 输出格式

### 4.1 家庭作业

松解与拉伸动作表格，会员在家自行完成，不占用私教课时间。

| 序号 | 动作 | 目标肌肉 | 原因 |
|-----|------|---------|------|
| 1 | ... | ... | ... |

不标注频次，会员根据自身能力量力而行。

### 4.2 课堂训练模块池

按优先级 P1 → P2 → P3 排序，不分部位模块，教练每节课根据会员状态灵活选取。

| 优先级 | 目标肌肉 | 为什么练 | 推荐方向 |
|-------|---------|---------|---------|
| P1 | ... | ... | ... |

每项仅包含：目标肌肉、原因、推荐方向。不指定具体动作名、组数和次数——普拉提器械多，动作丰富，具体动作由教练根据器械和会员状态灵活选择。

### 4.3 禁忌症（三级制）

| 分级 | 标签 | 含义 |
|-----|------|------|
| 🔴 绝对禁止 | 任何情况下都不可进行 | 如跳跃、急停变向、单腿负重（ACL术后） |
| 🟡 谨慎评估 | 满足严格前提条件后可引入 | 如单腿桥式（需医生许可+稳定性测试通过） |
| 🟢 安全可行 | 在正确执行下安全 | 如双腿对称桥式、仰卧颔首 |

以表格呈现：

| 分级 | 动作 | 为什么 |
|-----|------|--------|
| 🔴 | 单腿桥式 | 单腿支撑时ACL剪切力是双腿的3-4倍 |
| 🟡 | 靠墙微蹲 | 双膝ACL重建叠加下需确认膝关节稳定性 |
| 🟢 | 双腿对称桥式 | 仰卧位消除体重压缩负荷，负荷可控 |

### 4.4 特别注意事项

基于病史的额外提醒，如：

- "左腿ACL术后仅1年，移植物需18-24个月完全成熟，所有动作强度降低50%"
- "糖尿病患者肩周炎发病率是普通人的3-5倍，所有训练需更加保守"

## 5. 核心处理逻辑

### 5.1 关键词匹配

解析体态描述中的关键词，匹配知识库条目。匹配规则：

- 精确匹配条目名称（如"足弓塌陷" → flatfoot）
- 同义词映射（如"五十肩" → frozen-shoulder，"膝超伸" → knee-hyperextension）
- 部位匹配（如"胸椎" → 匹配所有 category 为"脊柱"的条目）

### 5.2 信息提取

从匹配条目的数据结构中提取：

- **紧张肌肉**：从 anatomy 第二段"缩短/紧张"部分提取
- **薄弱肌肉**：从 anatomy 第二段"薄弱/延长"部分提取
- **训练原因**：提取每个肌肉的"→"原因描述
- **推荐方向**：从 training.actions 中提取动作名称作为方向性参考
- **优先级**：从 anatomy 第三段"优先顺序"提取 P1/P2/P3/P4

### 5.3 去重合并

同一肌肉出现在多个条目中时，合并为一个条目，原因说明整合多个来源的描述。

例："小腿三头肌"同时出现在"足弓塌陷"和"习惯性崴脚"两个条目中。

合并后：
- 目标肌肉：小腿三头肌
- 原因：足弓塌陷者小腿三头肌常处于保护性紧张状态以维持站立稳定；踝关节扭伤后进入保护性紧张以限制活动。分段松解可恢复踝关节背屈和内翻活动度。

### 5.4 优先级排序

同一优先级的动作按以下规则排序：

1. P1（松解优先）→ 家庭作业部分
2. P1（激活优先）→ 课堂训练模块池
3. P2 → 课堂训练模块池
4. P3 → 课堂训练模块池

### 5.5 病史叠加

特殊病史对应的禁忌症和注意事项规则：

- ACL重建术后 N 个月：根据术后时间自动判定禁忌级别（术后<6个月为🔴，6-18个月为🟡，>18个月根据腿别区分）
- 糖尿病：自动叠加糖尿病相关注意事项
- 骨质疏松：自动叠加骨质疏松相关禁忌
- 孕产：自动叠加孕产相关禁忌

## 6. 数据结构

```typescript
interface PersonalizedPlanInput {
  memberId: string;
  conditions: string[];        // 体态标签
  conditionText?: string;      // 自然语言描述
  medicalHistory: string[];    // 病史标签
  medicalHistoryText?: string; // 病史自由输入
}

interface HomeworkItem {
  order: number;
  action: string;        // 具体动作描述
  targetMuscle: string;  // 目标肌肉
  reason: string;        // 原因
}

interface ClassModule {
  priority: number;      // 1 | 2 | 3
  targetMuscle: string;  // 目标肌肉
  reason: string;        // 为什么练
  direction: string;     // 推荐方向
  sourceIds: string[];   // 来源知识库条目ID
}

interface Contraindication {
  level: 'forbidden' | 'caution' | 'safe';  // 🔴 🟡 🟢
  action: string;        // 动作描述
  reason: string;        // 为什么
}

interface SpecialNote {
  note: string;          // 注意事项文本
}

interface PersonalizedPlan {
  id: string;
  memberId: string;
  title: string;
  createdAt: string;
  homework: HomeworkItem[];
  classModules: ClassModule[];
  contraindications: Contraindication[];
  specialNotes: SpecialNote[];
}
```

## 7. 前端交互设计

### 7.1 输入页面

1. **会员选择**：选择已有会员或新建会员
2. **体态描述**：
   - 标签云：所有知识库条目名称，点击多选
   - 文本框：自然语言补充描述
3. **特殊病史**：
   - 快捷标签：常见病史一键添加
   - 文本框：自由输入补充病史
4. **生成按钮**：点击后调用后端生成综合计划

### 7.2 预览/编辑页面

1. **家庭作业**：可增删行、修改内容
2. **课堂训练模块池**：可增删行、调整优先级、修改内容
3. **禁忌症**：三级制表格，可增删行
4. **特别注意事项**：可增删行
5. **操作按钮**：
   - 保存计划
   - 导出PDF
   - 返回修改输入

### 7.3 每节课的教练使用流程

1. 打开会员的综合计划
2. 根据当天会员状态，从模块池中选取合适的模块
3. 选取的模块可标记为"已授课"，记录日期和备注
4. 系统显示每个模块的授课历史

## 8. 后端接口设计

```
POST /api/personalized-plans
  输入: PersonalizedPlanInput
  输出: PersonalizedPlan

GET /api/personalized-plans/:id
  输出: PersonalizedPlan

PUT /api/personalized-plans/:id
  输入: Partial<PersonalizedPlan>
  输出: PersonalizedPlan

DELETE /api/personalized-plans/:id

GET /api/personalized-plans?memberId=xxx
  输出: { data: { id, title, createdAt }[] }
```

## 9. 测试策略

1. **单元测试**：
   - 关键词匹配逻辑（各种体态描述 → 正确匹配条目）
   - 信息提取逻辑（从 anatomy/training 正确提取肌肉、原因、优先级）
   - 去重合并逻辑（同一肌肉多来源 → 正确合并原因）
   - 病史叠加逻辑（ACL术后时间 → 正确分级禁忌症）

2. **集成测试**：
   - 端到端生成测试（输入 → 输出格式正确、内容合理）
   - 边界测试（无匹配条目、所有条目都匹配、空输入）

3. **人工校验**：
   - 由普拉提专业教练审核10-20份生成的计划，评估专业性和实用性

## 10. 实现优先级

第一阶段（MVP）：
1. 关键词匹配引擎（标签选择 + 精确匹配）
2. 信息提取与去重合并
3. 基础禁忌症叠加（ACL、糖尿病等常见病史）
4. 前端输入页面和预览页面
5. 后端存储接口

第二阶段：
1. 自然语言解析（体态描述文本 → 关键词提取）
2. 更丰富的病史规则库
3. 模块授课记录和历史追踪
4. PDF导出优化（排版美化）

第三阶段：
1. 计划效果追踪（教练标注模块完成情况和会员反馈）
2. 基于效果的智能调整建议
